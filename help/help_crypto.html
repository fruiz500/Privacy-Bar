Crypto & Privacy Help

Start Here: The Lock and Key Story
Privacy Bar is built on Asymmetric Encryption. To understand how it protects you, think of a physical padlock:

  • Your Key (Secret): This is your private digital identity, generated from your Master Key (MK). You must never share it. Only this Key can open data secured with your matching Lock.
  • Your Lock (Public): A string derived from your Key. You share this freely (via email, social media, etc.) so others can send you private messages.
  • The Logic: You encrypt data using the recipient's Lock. Once locked, only their Key can open it. Even you cannot unlock it once it has been encrypted.
How to Secure and Send a Message
  1. Enter your MK and email address.
  2. Select the recipient(s) in the Directory. Tip: Select "Me" to ensure you can read the message yourself later.
  3. Type your message or drag a file into the Input Box.
  4. Select a mode: Signed (identifies you), Anonymous, or one of the Pro modes.
  5. Click Encrypt to inject the result into the page, or To File (Pro) to download it.
How to Open and Verify a Message
  1. Ensure your MK and email are filled.
  2. Click an encrypted block on the webpage to load it, or drag an encrypted file into the Input Box.
  3. Click Decrypt. The app automatically detects the mode and sender; no manual selection is needed.
  4. Hidden Messages (Pro feature): If you suspect a "deniable" message is hidden inside, click the Hidden Msg button after the main decryption is successful.
Tip: you can decrypt data encrypted in a Pro mode, even at the free level.
Using Shared Keys (When No Recipient is Selected)
If you click Encrypt without selecting anyone in the Directory, the app follows this priority chain to choose a Symmetric (shared key) mode:
  1. Folder Key: Uses the Folder Key currently in memory.
  2. Pad Mode: Uses the entropy from a loaded Pad file.
  3. Shared Key Prompt: Prompts you for a key shared with the recipients.
    • Human Mode: Entering 5 words separated by commas triggers the PolyCrypt algorithm.
Note: If you cancel the Key prompt, the process is aborted to prevent accidental plaintext exposure. To invite a new user instead, use the "Invitation" link.
Future-Proofing: Post-Quantum (PQ) Encryption PRO
Post-Quantum (PQ) encryption protects your data against future threats from quantum computers that could potentially break traditional algorithms.
  • PQ Lock Requirement: To use PQ modes, recipients must have a compatible PQ Lock saved in the Directory.
  • Automatic Filtering: When you select a PQ encryption mode, the recipient list automatically filters to display only those contacts with a valid PQ lock.
  • Two Modes: PQ-encrypted data can be in PQ-Signed mode, where recipients can verify the sender, or PQ-Anonymous carrying zero sender information.
  • Legacy Support: PQ features are integrated without removing legacy functionality, maintaining compatibility with standard encryption methods.
Note: This feature requires a Pro license.
Unbreakable Security: Pad Mode & Entropy ExtractionPRO
Pad Mode is mathematically unbreakable by any computer, present or future.
  • Entropy Extraction: Unlike standard encryption, Pad Mode extracts the entropy (randomness) of a regular file to create a key.
  • Usage: Drag your key file to the area below the Action buttons. Ensure no recipients are selected and click Encrypt.
  • Requirement: Both parties must use the exact same file (content must match exactly).
Note: This feature requires a Pro license.
Human Mode: The PolyCrypt Algorithm
Human Mode uses the PolyCrypt cipher, designed to be computable manually with pencil and paper.
  • How to Trigger: Enter 5 words separated by commas when prompted for a Shared Key.
  • Full technical details and manual instructions: privacybar.net/human/.
Self-Destructing Messages (Read-once) PRO

Read-once is the highest level of digital privacy. Decryption keys are destroyed immediately after use, so a message can never be opened twice.

  • The Handshake: The first time you message someone, a "Reset Handshake" occurs. You will see a "Waiting" status until they reply.
  • Once per Message: As soon as you successfully decrypt a message, the extension updates your keys for the next one. Re-opening the same message will fail.
  • Fixing Sync Errors: If you see a "Sync Error," your extensions are out of step. Both you and the recipient should click Manage Directory, find the contact, and click Reset RO. The next message will re-sync your privacy.
Securing Files via Drag & DropPRO
You can secure entire files by dragging them into the Input Box:
  • Encryption: Drag any file into the box to encrypt it using your selected mode.
  • Decryption: Drag an encrypted file (or camouflaged image) into the box. The decrypted result is saved to your Downloads folder.
  • Camouflage Integration: If an image is already loaded, encrypted files are automatically camouflaged within that image.
Note: This feature requires a Pro license.
Image Secrets: Steganography & Camouflage
1. Loading an Image:
  • Click the image button at the bottom of the UI, or Alt+Click (Opt+Click on Mac) any image on a webpage.
2. LSB (Invisible Embedding): Hides data inside pixels. Load an image, type text, and click To Image or drag a file onto the image (Pro).

3. Camouflage (High Capacity): Appends data to the file. Happens automatically when encrypting to file if an image is loaded.

4. Extraction: Click From Image for LSB, or drag the image into the Main Box for Camouflage.

Note: The Camouflage feature requires a Pro license. LSB embedding is enabled in free mode
The Digital Handshake: Sending Invitations
To start communicating with someone new, use the Create Invitation link on the UI.
  • What it does: It generates a friendly "onboarding" message containing your public Lock and instructions for installing Privacy Bar.
  • No Encryption: Invitations are plaintext and contain no encrypted material. They are simply a way to introduce yourself and share your Lock.
  • Usage: Click the link to place the text into the page (or clipboard) for easy sending.
Managing Contacts, PQ Locks, and Fingerprints
The app manages both 50-character Classic Locks and much longer PQ Locks (Post-Quantum).
  • Verification: Once both Locks are entered for a contact, a 16-letter fingerprint appears next to their name for easy verification.
  • PQ Filtering: If you select a PQ encryption mode, the recipient list will automatically filter to show only users who have a PQ Lock saved.
  • Legacy Users ($): Names starting with $ are hidden from selection but remain active for recognizing incoming messages.
  • Lists: Create a =List= by entering names separated by commas (e.g., Alice,Bob) in the Directory manager.
Advanced: Folder Keys, Hidden Messages & P2P ChatPRO
  • Folder Keys: Click Encrypt with an empty Input Box to create one. You can re-encrypt them for new groups or click Wrap to hide them in a decoy "business email."
  • Split/Join: Divide a secret into parts (e.g., "3,2" where 3 parts are made and any 2 rebuild it). This is a mathematical split, not encryption.
  • P2P Chat: Create an invitation for a Jitsi chat. Decrypting the result provides a "Join" button and copies the E2E password to your clipboard.
Note: These features require a Pro license.
Deniable Privacy: The Hidden Message PRO

You can hide a secret "decoy" message inside a standard encrypted block.

  1. Toggle the Hidden Msg button before encrypting.
  2. When prompted, enter a different secret key than your Master Key.
  3. The extension will stretch this decoy key separately, creating a hidden layer that only someone with that specific key can find.

Note: If someone forces you to decrypt the main message, the hidden one remains invisible and mathematically undetectable.

Using Privacy Bar on Mobile

While most mobile browsers do not currently support Chrome extensions, you can still access your synthesized passwords and encrypt/decrypt messages on any device (basic modes only) thanks to the deterministic nature of our math.

Privacy Bar Mobile:
https://privacybar.net/mobile
  • Parity: Because the generation is mathematical, entering the same Master Key and Email on your phone will produce the exact same Lock and Key as the desktop extension.
  • Security: No data is sent to a server; all processing occurs locally in your mobile browser's memory.
  • Pro Tip: Open the link in Safari (iOS) or Chrome (Android) and use the "Add to Home Screen" option to use it like a standalone app.
