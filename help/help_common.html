PassLok & KyberLock Compatibility

Privacy Bar is designed for interoperability with our established cryptographic tools. However, compatibility depends on the Mode and Output Type.

✅ Fully Compatible (Text Messages):
  • PassLok series: Standard Anonymous and Signed text modes are fully compatible between both apps, as well as the hidden message feature. Locks are the same.
  • KyberLock series (PQ): Post-Quantum Anonymous and Signed modes are compatible. Note: Privacy Bar's PQ Signed mode is currently one-way compatible with KyberLock. Locks are compatible.
❌ Not Compatible (App-Specific):
  • File Output: Encrypted files (.pbx) and Image Camouflage are unique to Privacy Bar. They cannot be decrypted by PassLok or KyberLock, which still have their own formats for encrypted files.
  • Image Steganography: While all apps can hide data in pixels, Privacy Bar uses a different mathematical method (PRNG) to distribute the data. Messages hidden in images via Privacy Bar cannot yet be extracted by our other tools.

How to Activate Pro Status

Advanced features are unlocked via a Pro License. Once you have purchased your key via Lemon Squeezy and received it via email, follow these steps:

  1. Open the dialog: Click the Activate Pro link or any button with a lock on it to display the activation dialog.
  2. Activate: Paste your entire license key (including yoour name) and click Activate.
  • Local Verification: Activation happens entirely within your browser. The app does not "phone home" to a server to check your license once it is active.
  • Feature List: Pro status unlocks Post-Quantum modes, User-Provided Passwords, Page Sanitization, Camouflage, and Read-once Page Notes.

Tip: If the key doesn't work, ensure there are no accidental spaces at the beginning or end of your name/email.

Why is the app "Stretching" my key?

When you see the blinking "Stretching" message, Privacy Bar is performing thousands of complex mathematical operations to harden your Master Key.

  • Security: This is a "Master Key Shield." It makes it exponentially harder for automated programs to guess your password.
  • Performance: It is normal for the app to pause for a second during this process. This ensures that even a simple Master Key provides professional-grade protection.
How to Report a Bug or Security Vulnerability

Found a glitch? Since Privacy Bar is a solo project, your detailed reports help keep the build stable while the author is at his day job.

  1. Check the Console: If something isn't clicking, right-click the Side Panel, select Inspect, and click the Console tab. Note any red error messages.
  2. Gather Details: Note your Browser version, Operating System (Mac/Win/Linux), and the Extension version number.
  3. Submit the Report: Use the official Report Page to send your findings via Formspark.
⚠️ IMPORTANT: Never include your Master Key (MK), private keys, or actual sensitive notes in a bug report. Use dummy data ("test123") to demonstrate the issue.

For inquiries and support: https://privacybar.net/report.html

How to Authenticate a Lock (via Email)

When you receive a Lock (Public Key) from someone via email, you must ensure it hasn't been swapped by a "Man-in-the-Middle" attacker.

  1. Receive: Save the sender's Lock into your directory.
  2. Verify Fingerprint: Contact the sender through a different channel (Signal, Voice Call, or in person).
  3. Compare: Read the first and last 10 characters of the Lock string to them. If they match exactly, the Lock is authentic.
  4. If you are the one reading your Lock, it is the string displayed under name "Me" when you click Manage Directory in the Crypto tab.
  5. If you are authenticating a PQ Lock, compare the string that appears to the right of the name after clicking Manage Directory.

Note: If the Lock doesn't match, delete it immediately. Someone may be trying to intercept your communications.

Verify Code Integrity (Audit)
Privacy Bar is source-available on GitHub so you can verify that the code running in your browser matches the developer's original build. This facilitates security auditing and trust.

Official Repository:
https://github.com/fruiz500/Privacy-Bar

To verify the authenticity of the code after installation, do this:

1. Find your Extension Folder:
  • Mac: ~/Library/Application Support/Google/Chrome/Default/Extensions/objpfdibjogbdpjjpomnoaflkgdeaanb
  • Windows: %LOCALAPPDATA%\Google\Chrome\User Data\Default\Extensions/objpfdibjogbdpjjpomnoaflkgdeaanb
  • Linux: ~/.config/google-chrome/Default/Extensions/objpfdibjogbdpjjpomnoaflkgdeaanb
2. Download the Manifest: Save the official hashes.txt file into that folder.

3. Run the Verification Command:
MacOS/Linux:
shasum -a 256 -c hashes.txt

Windows (PowerShell):
Get-Content hashes.txt | Where-Object { $_ -notmatch '^#' } | ForEach-Object { $h, $f = $_ -split ' +'; if ((Get-FileHash $f).Hash -eq $h) { "$f: OK" } else { "$f: FAIL" } }
Troubleshooting Mismatches (LF vs CRLF):
If a file says FAIL but you haven't changed the code, it is likely due to line-ending conversion. Windows often converts LF to CRLF, which changes the file's hash. To be certain, inspect the file manually via DevTools (F12) to audit the raw logic.
